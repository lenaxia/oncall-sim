# docs/backlog

User stories and acceptance criteria organized by development phase.

## Rules

- One folder per phase, matching the phase numbering in `README-LLM.md`
- Each story file is self-contained — title, user story, acceptance criteria, tasks, status
- Update story status and task checkboxes as work progresses
- Do not start a phase without reading all its stories first

## Structure

- `01-shared-types/` — Phase 1 stories
- `02-metric-generator/` — Phase 2 stories
- `03-scenario-loader/` — Phase 3 stories
- `04-game-engine/` — Phase 4 stories
- `05-llm-client/` — Phase 5 stories
- `06-api/` — Phase 6 stories
- `07-ui-components/` — Phase 7 stories
- `08-sim-tabs/` — Phase 8 stories
- `09-coach-debrief/` — Phase 9 stories

## Story Status Values

- `Not Started`
- `In Progress`
- `Complete`
- `Blocked` — document the blocker in the story file
